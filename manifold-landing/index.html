First time here? Suggested tour

Four federated tools for managing the full lifecycle of software deployments at a mid-size engineering organization: a service catalog with a dependency graph (Groundwork), a team-staffing dashboard (Union), a deployment-planner with governance bylaws (Cityhall), and a test-environment coordinator (Yard), plus an advisory tier surfacing cross-federation concerns (Lobby). Each app reads from one federated graph; cross-app links and shared IDs work everywhere.

Demo data against a fictional logistics company, Meridian Freight Solutions — 30 microservices, 17 people across 5 teams, 100+ test runs, with the dependency cycles, deployment freezes, and stretched teams you'd expect to see in real production data. No authentication, no persistence guarantees; everything resets when the seed runs.
  1. Start in Groundwork. Browse the deployable catalog; the Graph tab gives you the dependency network. Nodes coloured by deployment status, edges by criticality. Click a deployable to see its services, dependents, test environments, and a focused subgraph of its neighbourhood.
  2. Follow a deployable's team badge to Union. From the Work board, click a card's cr: link to follow the work order into Cityhall as a change request.
  3. In Cityhall, the Org tab shows the bylaw cascade through the Meridian hierarchy. The Plans tab renders a deployment Gantt for each change request — including, where applicable, a highlighted graph of any dependency cycle the planner found.
  4. In Yard, browse test environments by cost and spinup time, follow a recent test run back to the change request that triggered it, or open Lifecycle for data freshness and the reset planner.
  5. Lobby aggregates concerns derived from the federation — blocked upstreams, dependency cycles, undocumented interfaces, missing environments — with an append-only lifecycle and saved views.
Apps
Groundwork Service catalog — deployables, services, dependencies, contracts, and SLAs, with an interactive dependency graph. Union Team staffing — people, capacity in story points, and a kanban work board linking to deployables and change requests. Cityhall Deployment planning — org hierarchy, cascading bylaws (Approval / Window / Quiesce / Freeze gates), Mermaid Gantt output. Yard Test coordination — environments, infrastructure cost, mock and data sources, data syncs, lifecycle / reset planner, and run history. Lobby Advisory tier — derived concerns across the federation (blocked upstreams, dependency cycles, undocumented interfaces, schedule contention) with append-only lifecycle.
